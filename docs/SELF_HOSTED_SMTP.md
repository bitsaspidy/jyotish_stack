# Self-Hosted SMTP for Jyotish Stack AI

This guide configures domain mail on the same Ubuntu VPS as the app, without Hostinger Email, Gmail, Brevo, or SendGrid.

The project domain currently used by the repo is `jyotishstack.com`. If you intentionally own/use `jyotisstack.com` without the `h`, pass that domain to the setup script instead and update `server/.env` accordingly.

## Mailboxes

The app already routes mail to these three domain accounts:

| Mailbox | Purpose |
| --- | --- |
| `sales@jyotishstack.com` | Sales inquiries |
| `team@jyotishstack.com` | Support/general inquiries, newsletters, admin custom email |
| `account@jyotishstack.com` | Auth emails, password resets, account/billing, subscription receipts |

## DNS Before Running The Script

Create these records in DNS:

| Type | Name | Value |
| --- | --- | --- |
| `A` | `mail` | VPS public IP |
| `MX` | `@` | priority `10`, target `mail.jyotishstack.com` |
| `TXT` | `@` | `"v=spf1 mx -all"` |
| `TXT` | `_dmarc` | `"v=DMARC1; p=quarantine; rua=mailto:account@jyotishstack.com; adkim=s; aspf=s"` |

After the script runs, copy the DKIM record from:

```bash
sudo cat /etc/opendkim/keys/jyotishstack.com/mail.txt
```

Also set reverse DNS/PTR for the VPS public IP to:

```text
mail.jyotishstack.com
```

Without correct PTR, Gmail/Outlook delivery will be poor even when SMTP works.

## Firewall

For self-hosted mail, the VPS firewall must allow:

| Port | Purpose |
| --- | --- |
| `22/tcp` | SSH |
| `25/tcp` | Server-to-server SMTP receiving and outbound delivery |
| `80/tcp` | HTTP and Let's Encrypt |
| `443/tcp` | HTTPS |
| `587/tcp` | Authenticated SMTP submission from the app/email clients |
| `993/tcp` | IMAPS for reading mail in an email client |

Do not expose MySQL `3306`, Next.js `3000`, Express `5000`, or phpMyAdmin `8081`.

## Install Mail Server

Run this on the VPS after `mail.jyotishstack.com` points to the VPS:

```bash
cd /var/www/jyotish-stack
git pull --ff-only origin main
sudo bash scripts/setup-self-hosted-smtp.sh jyotishstack.com
```

The script installs and configures:

- Postfix for SMTP and authenticated submission.
- Dovecot for IMAP and SMTP authentication.
- OpenDKIM for DKIM signing.
- A Let's Encrypt certificate for `mail.jyotishstack.com`.
- The `sales@`, `team@`, and `account@` mailboxes.

Mailbox passwords are generated once and stored root-only:

```bash
sudo cat /root/jyotish-mailbox-passwords.txt
```

DNS notes generated by the script are stored root-only:

```bash
sudo cat /root/jyotish-mail-dns-records.txt
```

## App Environment

Copy the generated passwords into `/var/www/jyotish-stack/server/.env`:

```env
SMTP_HOST=mail.jyotishstack.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_REQUIRE_TLS=true
SMTP_TLS_REJECT_UNAUTHORIZED=true
SMTP_TLS_SERVERNAME=mail.jyotishstack.com

MAIL_SALES_USER=sales@jyotishstack.com
MAIL_SALES_PASS=<password from /root/jyotish-mailbox-passwords.txt>
MAIL_SALES_FROM=Jyotish Stack Sales <sales@jyotishstack.com>

MAIL_TEAM_USER=team@jyotishstack.com
MAIL_TEAM_PASS=<password from /root/jyotish-mailbox-passwords.txt>
MAIL_TEAM_FROM=Jyotish Stack Support <team@jyotishstack.com>

MAIL_ACCOUNT_USER=account@jyotishstack.com
MAIL_ACCOUNT_PASS=<password from /root/jyotish-mailbox-passwords.txt>
MAIL_ACCOUNT_FROM=Jyotish Stack <account@jyotishstack.com>

SMTP_USER=account@jyotishstack.com
SMTP_PASS=<account mailbox password>
SMTP_FROM=Jyotish Stack <account@jyotishstack.com>
```

Restart the API after editing `server/.env`:

```bash
cd /var/www/jyotish-stack
pm2 restart jyotish-api --update-env
pm2 save
```

## Validate

Check services and listeners:

```bash
sudo systemctl status postfix dovecot opendkim --no-pager
sudo ss -ltnp | grep -E ':(25|587|993)\b'
```

Check SMTP authentication:

```bash
swaks --to account@jyotishstack.com \
  --from account@jyotishstack.com \
  --server mail.jyotishstack.com \
  --port 587 --tls \
  --auth LOGIN \
  --auth-user account@jyotishstack.com \
  --auth-password '<account password>'
```

Check outbound delivery after PTR, SPF, DKIM, and DMARC are live:

```bash
swaks --to your-test-gmail@gmail.com \
  --from account@jyotishstack.com \
  --server mail.jyotishstack.com \
  --port 587 --tls \
  --auth LOGIN \
  --auth-user account@jyotishstack.com \
  --auth-password '<account password>'
```

Watch logs:

```bash
sudo tail -f /var/log/mail.log
```

## Important Deliverability Notes

Self-hosted email is sensitive to DNS and IP reputation:

- Outbound port `25` must be allowed by the VPS provider.
- PTR/reverse DNS must match `mail.jyotishstack.com`.
- SPF, DKIM, and DMARC must pass.
- New VPS IPs may need warm-up before bulk newsletters.
- Keep newsletters modest at first and watch bounces in `/var/log/mail.log`.
